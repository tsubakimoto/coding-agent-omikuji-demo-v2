# テスト仕様書

このディレクトリには、Omikuji（おみくじ）アプリケーションのPlaywrightテストが含まれています。

## テストファイル構成

### `basic.spec.js`
- アプリケーションの基本的な読み込み機能
- ページタイトルの確認
- 主要要素の表示確認
- 初期状態の検証

### `fortune-drawing.spec.js`
- おみくじ抽選機能のテスト
- ローディング状態の確認
- 運勢結果の表示
- ランダム性の検証
- 色分け表示の確認

### `theme-toggle.spec.js`
- ライト/ダークテーマ切り替え機能
- テーマアイコンの変更
- アクセシビリティ（title属性）
- テーマ設定の永続化

### `history.spec.js`
- 履歴機能のテスト
- 履歴項目の保存と表示
- 5件制限の確認
- タイムスタンプ表示
- ローカルストレージ永続化

### `responsive.spec.js`
- レスポンシブデザインのテスト
- モバイル、タブレット、デスクトップ対応
- 各画面サイズでの機能確認
- レイアウトの適応性

### `ui-animation.spec.js`
- UIアニメーションとエフェクト
- ローディングアニメーション
- ホバーエフェクト
- ボタンの無効化/有効化
- グラデーション背景

### `accessibility.spec.js`
- アクセシビリティ機能
- 見出し構造の確認
- キーボードナビゲーション
- スクリーンリーダー対応
- カラーコントラスト

## テスト実行方法

### 基本実行
```bash
npm run test
```

### UI モードでの実行（開発時）
```bash
npm run test:ui
```

### ヘッド付きブラウザで実行（デバッグ時）
```bash
npm run test:headed
```

### テストレポートの表示
```bash
npm run test:report
```

## テスト設定

- **ベースURL**: `http://localhost:4173` (Viteのプレビューサーバー)
- **ブラウザ**: Chromium、Firefox、WebKit
- **モバイル**: Pixel 5、iPhone 12
- **並列実行**: 有効
- **リトライ**: CI環境で2回

## 注意事項

- テストを実行する前に `npm run build` でアプリケーションをビルドしてください
- Playwrightが自動的にプレビューサーバーを起動します
- 各テストは独立して実行され、状態がクリアされます
- ローカルストレージのテストでは明示的にクリアしています
